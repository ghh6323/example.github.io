## 前言

在 Django 中，用户通过浏览器访问网页时，会发生两件事：

1. **请求（Request）**：浏览器把用户的数据发给服务器
2. **响应（Response）**：服务器处理完后，把结果返回给浏览器

今天用最简单的方式，把这套流程讲清楚。

---

## 一、Request 是什么

`request` 是一个**对象**，里面封装了用户通过浏览器发送过来的**所有数据**。

想象 `request` 是一个快递包裹，浏览器把各种信息打包塞进去，Django 收到后拆开来看。

---

## 二、Request 能拿到什么

### 1. 请求方式：GET 还是 POST

```python
def something(request):
    print(request.method)   # 输出 GET 或 POST
```

| 方式 | 什么时候用 |
|------|-----------|
| **GET** | 打开网页、搜索、翻页（从服务器要数据） |
| **POST** | 提交表单、登录、注册（往服务器传数据） |

---

### 2. URL 上的参数

用户在地址栏输入：

```
/something/?n1=123&n2=999
```

Django 这样取：

```python
def something(request):
    print(request.GET)              # 全部参数：{'n1': ['123'], 'n2': ['999']}
    print(request.GET.get("n1"))    # 单个取值：123
```

> `?` 后面跟着的就是 URL 参数，用 `&` 连接多个。

---

### 3. 请求体里的数据

用户提交表单时，数据藏在请求体里，不在 URL 上显示：

```python
def something(request):
    print(request.POST)                  # 全部表单数据
    print(request.POST.get("username"))  # 取单个字段
```

> **GET 的数据在 URL 上（看得见），POST 的数据在请求体里（看不见）。**

---

## 三、Response 是什么

处理完请求后，Django 需要**返回点什么**给浏览器。有三种方式：

---

### 方式 1：返回纯文本（HttpResponse）

最简单，直接返回一串文字：

```python
from django.shortcuts import HttpResponse

def something(request):
    return HttpResponse("你好，这是返回的内容")
```

浏览器显示：

```
你好，这是返回的内容
```

---

### 方式 2：返回 HTML 页面（render）

读取模板文件，把数据填进去，再返回完整的 HTML：

```python
from django.shortcuts import render

def something(request):
    return render(request, 'something.html', {"title": "来了"})
```

**something.html：**

```html
<h1>{{ title }}</h1>
```

浏览器显示：

```
来了
```

> `{"title": "来了"}` 这个字典，把 `"title"` 这个钥匙传给模板，模板用 `{{ title }}` 开锁取值。

---

### 方式 3：跳转到别的页面（redirect）

不显示内容，直接让浏览器去另一个地址：

```python
from django.shortcuts import redirect

def something(request):
    return redirect("https://www.baidu.com")
```

用户访问这个页面，瞬间被带到百度。

---

## 四、完整代码对比

```python
from django.shortcuts import HttpResponse, render, redirect

def something(request):
    # ========== 请求部分：从浏览器拿数据 ==========

    # 1. 看是什么请求方式
    print(request.method)

    # 2. 拿 URL 上的参数  /something/?n1=123
    print(request.GET.get("n1"))

    # 3. 拿表单提交的数据
    print(request.POST.get("username"))

    # ========== 响应部分：给浏览器返回数据 ==========

    # 4. 返回纯文字
    # return HttpResponse("返回内容")

    # 5. 返回渲染好的 HTML 页面
    # return render(request, 'something.html', {"title": "来了"})

    # 6. 跳转到其他网址
    return redirect("https://www.baidu.com")
```

## 六、GET vs POST 一张表看懂

| 对比项 | GET | POST |
|--------|-----|------|
| 数据放哪 | URL 上（`?key=value`） | 请求体里（看不见） |
| 获取方式 | `request.GET` | `request.POST` |
| 用途 | 搜索、翻页、打开页面 | 登录、注册、提交表单 |
| 安全性 | 低（数据暴露在地址栏） | 高（数据隐藏） |
| 数据大小 | 小（URL 长度有限） | 大 |

## 七、实战：登录表单

上面讲了原理，现在用登录功能串起来，看看请求和响应是怎么配合的。

### 7.1 登录页面（HTML）

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>登录</title>
</head>
<body>
    <h1>用户登录</h1>

    <form method="post" action="/login/">
        {% csrf_token %}

        <input type="text" name="user" placeholder="请输入用户名">
        <input type="password" name="pwd" placeholder="请输入密码">
        <input type="submit" value="提交">
    </form>
</body>
</html>
```

### 7.2 三个关键属性

| 属性 | 作用 | 类比 |
|------|------|------|
| `method="post"` | 数据放请求体里，不在 URL 暴露 | 快递保密件 |
| `action="/login/"` | 数据提交到哪个地址 | 快递收件地址 |
| `{% csrf_token %}` | Django 安全令牌，防止伪造攻击 | 快递签收验证码 |

> 去掉 `{% csrf_token %}`，Django 会直接返回 `403 Forbidden`，这是安全机制。

---

### 7.3 后端接收数据（views.py）

```python
from django.shortcuts import render, HttpResponse, redirect

def login(request):
    # GET 请求：用户打开登录页面
    if request.method == "GET":
        return render(request, "login.html")

    # POST 请求：用户点击了提交按钮
    # 从请求体里取表单数据
    username = request.POST.get("user")
    password = request.POST.get("pwd")

    print(f"收到用户名：{username}，密码：{password}")

    # 简单的验证逻辑（实际应该查数据库）
    if username == "admin" and password == "123456":
        # 登录成功，跳转到首页
        return redirect("/index/")
    else:
        # 登录失败，返回错误提示
        return render(request, "login.html", {"error": "用户名或密码错误"})
```

---

### 7.4 显示错误提示（login.html 改进）

```html
<form method="post" action="/login/">
    {% csrf_token %}

    <!-- 如果有错误，显示出来 -->
    <div style="color: red;">{{ error }}</div>

    <input type="text" name="user" placeholder="请输入用户名">
    <input type="password" name="pwd" placeholder="请输入密码">
    <input type="submit" value="提交">
</form>
```

> `{{ error }}` 就是后端传过来的 `"用户名或密码错误"`，模板语法自动渲染。

---

### 7.5 配置路由（urls.py）

```python
from django.urls import path
from app import views

urlpatterns = [
    path("login/", views.login),
    path("index/", views.index),
]
```

---

### 7.6 完整数据流向

```
用户打开 /login/
    ↓
Django 发现是 GET 请求
    ↓
返回 login.html（空表单）
    ↓
用户填写用户名、密码，点击提交
    ↓
浏览器打包数据：
  - method: POST
  - action: /login/
  - csrf_token: xxxxxx
  - user: admin
  - pwd: 123456
    ↓
Django 接收 POST 请求
    ↓
request.POST.get("user") → "admin"
request.POST.get("pwd") → "123456"
    ↓
验证通过 → redirect("/index/")  跳转首页
验证失败 → render(request, "login.html", {"error": "..."})  显示错误
```

---

## 八、一句话总结

> **Request**：浏览器 → Django，带着数据来。  
> **Response**：Django → 浏览器，带着结果回。

| 你想做什么 | 用什么 |
|-----------|--------|
| 返回一串文字 | `HttpResponse("文字")` |
| 返回一个网页 | `render(request, "模板.html", {"key": value})` |
| 跳转到别的页面 | `redirect("网址")` |
| 提交表单数据 | `method="post"` + `request.POST.get("字段名")` |
