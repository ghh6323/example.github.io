# 5.Django SSO 单点登录 · 从 0 手把手教程

> 场景：一个代码商店，用来购买代码。
> 首页：www.codeshop.com，VIP 系统：vip.codeshop.com，购物车系统：cart.codeshop.com，登录系统：login.codeshop.com。
> 目标：在登录系统登录一次，其余子系统自动识别，全站通行；一处退出，全站失效。

## 整体思路（30 秒看懂）

```
一个 Django 项目 + 四个 app，每个 app 对应一个子系统
        ↓
中间件：看你访问的是哪个子域名，就把请求交给对应的 app 处理
        ↓
登录时 Django 写入 sessionid Cookie，把它的 Domain 设成 .codeshop.com
        ↓
所有子系统共享这张 Cookie → 登录一次，全站通行
```

`sessionid` 就是需求里说的 **token**；`SESSION_COOKIE_DOMAIN = '.codeshop.com'` 就是需求里说的 **"cookie 的域要是所有子系统相同的域"**。

## 第 1 步：改 hosts 文件，让域名指向本机

本地没有真实的 `codeshop.com` 域名，用 hosts 文件"骗"浏览器，把 4 个域名都指向自己电脑。

用**管理员身份**打开记事本，编辑 `C:\Windows\System32\drivers\etc\hosts`，末尾加 4 行：

```
127.0.0.1  www.codeshop.com
127.0.0.1  vip.codeshop.com
127.0.0.1  cart.codeshop.com
127.0.0.1  login.codeshop.com
```

保存后，浏览器访问这些域名就会打到本机的 Django 服务。

```python
# ============ SSO 核心配置 ============
# Session Cookie 的域设为公共父域：所有 *.codeshop.com 子系统都能读到这张"通行证"
SESSION_COOKIE_DOMAIN = '.codeshop.com'
SESSION_COOKIE_AGE = 60 * 60 * 24 * 7   # 有效期 7 天

# 统一登录页地址（供登录校验装饰器跳转用）
LOGIN_PAGE_URL = 'http://login.codeshop.com:8000/'
```

> `SESSION_COOKIE_DOMAIN = '.codeshop.com'` 是整套系统的灵魂。没有它，登录系统写的 Cookie 别的子系统根本读不到。

## 第 4 步：写中间件（子域名 → 子系统的"分流器"）

在 `codeshop_sso/` 目录下**新建 `middleware.py`**：

```python
class SubdomainURLMiddleware:
    """根据访问的子域名，把请求分发给不同的子系统（app）"""

    SUBDOMAIN_URLCONFS = {
        'www': 'home.urls',        # www.codeshop.com   → home 应用
        'vip': 'vip.urls',         # vip.codeshop.com   → vip 应用
        'cart': 'cart.urls',       # cart.codeshop.com  → cart 应用
        'login': 'accounts.urls',  # login.codeshop.com → accounts 应用（注意拼写！）
    }

    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        host = request.get_host().split(':')[0]      # 去掉端口号
        subdomain = host.split('.')[0]               # 取第一段作为子域名，[0] 不能漏！
        # 动态指定本次请求使用哪个 app 的 urls.py
        request.urlconf = self.SUBDOMAIN_URLCONFS.get(subdomain, 'home.urls')
        return self.get_response(request)
```

## 第 5 步：写登录校验装饰器（未登录就送去登录系统）

在 `codeshop_sso/` 目录下**新建 `decorators.py`**：

```python
from functools import wraps
from urllib.parse import urlencode
from django.shortcuts import redirect
from django.conf import settings


def sso_login_required(view_func):
    """未登录 -> 跳转到统一登录系统，并带上 next 参数（登录完跳回来）"""
    @wraps(view_func)
    def wrapper(request, *args, **kwargs):
        if not request.user.is_authenticated:
            next_url = request.build_absolute_uri()  # 用户原本想访问的地址
            login_url = f"{settings.LOGIN_PAGE_URL}?{urlencode({'next': next_url})}"
            return redirect(login_url)
        return view_func(request, *args, **kwargs)
    return wrapper
```

## 第 6 步：写登录系统（accounts）

**`accounts/views.py`**（整个文件替换）：

```python
from django.contrib.auth import authenticate, login, logout
from django.shortcuts import render, redirect
from django.http import HttpResponse


def login_view(request):
    """统一登录入口：login.codeshop.com"""
    if request.method == 'POST':
        username = request.POST.get('username')
        password = request.POST.get('password')
        next_url = request.POST.get('next') or 'http://www.codeshop.com:8000/'

        user = authenticate(request, username=username, password=password)
        if user is not None:
            login(request, user)   # 关键！写入 session，浏览器得到 Domain=.codeshop.com 的 Cookie
            return redirect(next_url)
        return render(request, 'accounts/login.html', {'error': '账号或密码错误', 'next': next_url})

    next_url = request.GET.get('next', 'http://www.codeshop.com:8000/')
    return render(request, 'accounts/login.html', {'next': next_url})


def logout_view(request):
    """统一退出：所有子系统一起退出"""
    logout(request)
    return HttpResponse('<h2>已退出登录，所有子系统同时失效</h2>'
                        '<a href="http://www.codeshop.com:8000/">返回首页</a>')
```

**新建 `accounts/urls.py`**：

```python
from django.urls import path
from . import views

urlpatterns = [
    path('', views.login_view, name='login'),
    path('logout/', views.logout_view, name='logout'),
]
```

**新建登录页模板 `accounts/templates/accounts/login.html`**（注意是 `templates/accounts` **两层文件夹**）：

## 第 9 步：浏览器验证（见证 SSO 的时刻）

1. 先访问 `http://cart.codeshop.com:8000/`（购物车）
   → **自动跳到** `http://login.codeshop.com:8000/?next=...`，因为没登录
2. 输入账号密码登录
   → 登录成功，**自动跳回购物车**，显示你的用户名
3. 打开新标签访问 `http://vip.codeshop.com:8000/`（VIP 系统）
   → **没有让你再登录**，直接显示"你已被 SSO 自动识别"
4. 访问 `http://www.codeshop.com:8000/`
   → 首页显示"你好，xxx！"
5. 点"退出登录"，再访问 VIP
   → 又被踢回登录页，**一次退出，全站失效**

**看证据**：登录后按 `F12` → 应用（Application）→ Cookies，能看到 `sessionid` 的 Domain 是 **`.codeshop.com`**——这就是四个系统能共享登录状态的原因。

## 需求 → 代码对照表

| 需求原文 | 实现方式 |
|----------|----------|
| 跳转到登录系统完成登录 | `sso_login_required` 装饰器，302 跳 `login.codeshop.com` 并带 `next` |
| 登录后写入一个 cookie 保存 token | `login(request, user)` 写入 `sessionid` Cookie（Django 会话即 token） |
| 其他子系统要能访问到这个 cookie | `SESSION_COOKIE_DOMAIN = '.codeshop.com'` |
| 其他子系统携带 cookie 完成登录 | Django 的 Session + Auth 中间件自动校验，`request.user` 直接可用 |
| 四个子系统 | 一个项目 + 4 个 app + 子域名中间件分流 |

## 常见坑汇总（实测踩过的）

### 坑 1：浏览器显示 "A server error occurred. Please contact the administrator."

**原因**：`settings.py` 里 `DEBUG = False`，Django 把真实报错藏起来了。
**解决**：开发阶段保持 `DEBUG = True`，改完刷新就能看到黄色详细报错页。真正的 Traceback 也可以在 PyCharm 底部的"运行（Run）"窗口里看（浏览器 F12 控制台只能看到 500 状态码，没有细节）。

### 坑 2：`TypeError: unhashable type: 'list'`（middleware.py 报错）

**原因**：`subdomain = host.split('.')` 漏了 `[0]`。`split()` 返回的是列表 `['cart', 'codeshop', 'com']`，列表不能当字典的键。
**解决**：改成 `subdomain = host.split('.')[0]`。
**经验**：`str.split()` 永远返回列表，取其中一段必须用下标。

### 坑 3：`ModuleNotFoundError: No module named 'account'`

**原因**：app 叫 `accounts`（带 s），middleware.py 字典里写成了 `account.urls`（不带 s）。
**解决**：改成 `'login': 'accounts.urls'`。
**经验**：`ModuleNotFoundError` 九成是名字打错。保证文件夹名、INSTALLED_APPS、middleware 字典里的拼写**逐字母一致**。

### 坑 4：`TemplateDoesNotExist: accounts/login.html`

**原因**：模板放错层级，直接放在了 `accounts/templates/login.html`。
**解决**：必须是两层 `accounts/templates/accounts/login.html`。

### 坑 5：改了 hosts 文件没生效

**解决**：`Win+R` 输入 `cmd` 打开命令行，执行 `ipconfig /flushdns`，再重启浏览器。

### 坑 6：登录后还是被踢回登录页

**原因**：99% 是 `SESSION_COOKIE_DOMAIN` 没配置，或写成了 `codeshop.com`（少了前面的点）。
**解决**：检查 settings.py，必须是 `SESSION_COOKIE_DOMAIN = '.codeshop.com'`。

## 扩展方向（学完这套之后）

- 加注册功能：新建 `register_view`，用 `User.objects.create_user()`
- 改用 JWT 代替 session（前后端分离场景）
- 不同主域名之间的 SSO：学习 CAS 协议的 ticket（票据）机制
- Cookie 安全加固：`SESSION_COOKIE_SECURE = True`（需 HTTPS）、`SESSION_COOKIE_HTTPONLY = True`
